## рҹ“қ Refined Implementation Plan (Plan-First)                                  
                                                                                
### 1. Cropped RLE Preprocessing (`regions.py`)                                 
Currently, RLE encoding full-canvas masks for hundreds of regions causes        
massive binary bloat. We will refactor `regions.py` to only encode the          
**bounding box (bbox)** of each region.                                         
* **Logic:** Use `cv2.findNonZero` on the mask, then `cv2.boundingRect` to      
obtain `[x, y, w, h]`.                                                          
* **Slicing:** Slice the numpy mask: `cropped = mask[y:y+h, x:x+w]`.            
* **Metadata:** Store the `x, y` offset in `metadata.json` for every region.    
This ensures the RLE stream is minimal, reducing the DO storage footprint and   
network transfer.                                                               
                                                                                
### 2. WGSL Coordinate Mapping & Uniform Alignment                              
WebGPU uniform buffers require strict **16-byte alignment** for structs. We     
will define a `RegionMetadata` struct that allows the GPU to map the local      
cropped RLE coordinates back to the global mural space.                         
                                                                                
**WGSL Struct Definition:**                                                     
```wgsl                                                                         
struct RegionMetadata {                                                         
    offset: vec2<f32>,   // [x, y] from cv2.boundingRect (8 bytes)              
    size: vec2<f32>,     // [w, h] of the cropped mask (8 bytes)                
    color: vec4<f32>,    // RGBA (16 bytes)                                     
    // Total: 32 bytes (multiple of 16, no extra padding needed)                
};                                                                              
```                                                                             
* **Mapping Formula:** `global_pos = local_pos + offset`.                       
                                                                                
### 3. O(1) ID Detection (`useMuralController.ts`)                              
To trigger **shadcn/ui** tooltips and components without React re-render loops  
during mouse movement:                                                          
* **Strategy:** Utilize the `id_map.png` (where each pixel's color corresponds  
to a Region ID).                                                                
* **Implementation:** On `mousemove`, sample the `id_map` texture using an      
`ImageBitmap` or `Uint8Array` copy kept in a **Ref** (not State).               
* **Trigger:** Only update the React state when the detected `hoveredId`        
changes, minimizing virtual DOM reconciliations.

## вң… Verification: The "Grill"                                                 
                                                                                
* **Off-by-One Bbox Errors:** OpenCV's `boundingRect` returns `(x, y, w, h)`    
where `w` and `h` are counts. When slicing in Python (`mask[y:y+h, x:x+w]`),    
the stop index is exclusive, which matches the count. However, the WGSL mapping 
 must ensure `local_pos` (0 to 1) is correctly scaled by `size` before the      
`offset` is added to prevent edge-case "bleeding" of masks.                     
* **Streaming Race Conditions:** If the frontend requests multiple RLE masks    
from the Backend Worker simultaneously, we must ensure the `MuralStateObject`   
(Durable Object) uses `transactional` storage or individual `fetch` concurrency 
 limits to prevent memory spikes in the 128MB isolate.                          
* **GPUBuffer Lifecycle:** Rapid hovering could lead to hundreds of small       
GPUBuffer allocations. We will implement a **Buffer Pool** in                   
`useMuralController.ts` to reuse `GPUBuffer` objects for region highlight masks 
 rather than destroying/recreating them on every frame.                         
* **Sidecar Config Check:** The root `wrangler.toml` (Pages) must remove all    
`[[migrations]]` and `main` fields; these *must* reside in                      
`worker/wrangler.toml` to avoid the "Configuration file for Pages projects does 
 not support migrations" error.

    offset: vec2<f32>,   // [x, y] from cv2.boundingRect (8 bytes)              
    size: vec2<f32>,     // [w, h] of the cropped mask (8 bytes)                
    color: vec4<f32>,    // RGBA (16 bytes)                                     
    // Total: 32 bytes (multiple of 16, no extra padding needed)                
};                                                                              
```                                                                             
* **Mapping Formula:** `global_pos = local_pos + offset`.                       
                                                                                
### 3. O(1) ID Detection (`useMuralController.ts`)                              
To trigger **shadcn/ui** tooltips and components without React re-render loops  
during mouse movement:                                                          
* **Strategy:** Utilize the `id_map.png` (where each pixel's color corresponds  
to a Region ID).                                                                
* **Implementation:** On `mousemove`, sample the `id_map` texture using an      
`ImageBitmap` or `Uint8Array` copy kept in a **Ref** (not State).               
* **Trigger:** Only update the React state when the detected `hoveredId`        
changes, minimizing virtual DOM reconciliations.

## вң… Verification: The "Grill"                                                 
                                                                                
* **Off-by-One Bbox Errors:** OpenCV's `boundingRect` returns `(x, y, w, h)`    
where `w` and `h` are counts. When slicing in Python (`mask[y:y+h, x:x+w]`),    
the stop index is exclusive, which matches the count. However, the WGSL mapping 
 must ensure `local_pos` (0 to 1) is correctly scaled by `size` before the      
`offset` is added to prevent edge-case "bleeding" of masks.                     
* **Streaming Race Conditions:** If the frontend requests multiple RLE masks    
from the Backend Worker simultaneously, we must ensure the `MuralStateObject`   
(Durable Object) uses `transactional` storage or individual `fetch` concurrency 
 limits to prevent memory spikes in the 128MB isolate.                          
* **GPUBuffer Lifecycle:** Rapid hovering could lead to hundreds of small       
GPUBuffer allocations. We will implement a **Buffer Pool** in                   
`useMuralController.ts` to reuse `GPUBuffer` objects for region highlight masks 
 rather than destroying/recreating them on every frame.                         
* **Sidecar Config Check:** The root `wrangler.toml` (Pages) must remove all    
`[[migrations]]` and `main` fields; these *must* reside in                      
`worker/wrangler.toml` to avoid the "Configuration file for Pages projects does 
 not support migrations" error.
